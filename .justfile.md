# Justfile Quick Reference

This project uses [just](https://github.com/casey/just) as a command runner for local development automation.

## Installation

```bash
# macOS
brew install just

# Linux
cargo install just

# or download from https://github.com/casey/just/releases
```

## Quick Start

```bash
# Show all available commands
just

# Run all CI checks locally (emulates GitHub Actions)
just ci

# Quick pre-commit checks (fast)
just quick-check

# Full checks before pushing
just full-check
```

## Most Common Commands

### Quality Checks

```bash
just ci                    # Run all CI checks (emulates GitHub Actions)
just quick-check          # Fast checks (ruff lint + format check)
just lint                 # All linters (ruff, mypy, bandit, markdownlint)
just fix                  # Auto-fix all fixable issues
just lint-stats           # Show lint issue statistics
```

### Individual Checks

```bash
just ruff-lint            # Run Ruff linting
just ruff-format-check    # Check code formatting
just mypy                 # Type checking
just bandit               # Security checks
just isort-check          # Check import sorting
just markdownlint         # Markdown linting
```

### Auto-fixing

```bash
just fix                  # Fix all (isort + ruff)
just ruff-fix             # Fix ruff issues
just isort-fix            # Fix import sorting
just ruff-format          # Format code
```

### Testing

```bash
just test                 # Run all tests
just test-coverage        # Run with coverage report
just test-verbose         # Verbose test output
just test-file FILE       # Run specific test file
just test-pattern NAME    # Run tests matching pattern
just watch                # Watch mode (requires entr)
```

### Building & Publishing

```bash
just build                # Build distribution packages
just build-check          # Build and validate
just publish              # Publish to PyPI
just publish-test         # Publish to TestPyPI
```

### Setup & Installation

```bash
just setup                # Complete dev environment setup
just install-dev          # Install package in editable mode
just install-deps         # Install dev dependencies
just install-hooks        # Install pre-commit hooks
just update-hooks         # Update pre-commit hooks
```

### Development

```bash
just dashboard            # Run dashboard (non-TUI)
just dashboard-tui        # Run dashboard (TUI)
just help                 # Show merge-god CLI help
just validate-config      # Validate config.yaml
```

### Cleanup

```bash
just clean                # Clean build artifacts
just clean-all            # Clean everything including DB
```

### Git Helpers

```bash
just status               # Git status
just log                  # Recent commits
just branch NAME          # Create new branch
```

## Workflows

### Before Committing

```bash
# Option 1: Quick check (fast)
just quick-check

# Option 2: Full check with tests
just full-check

# Option 3: Fix and lint
just pre-commit
```

### Continuous Development

```bash
# Watch mode - auto-run tests on changes
just watch

# In another terminal
# Make changes...
# Tests run automatically
```

### Before Pushing

```bash
# Run full CI locally
just ci

# If all passes, you're good to push!
git push
```

### Release Process

```bash
# 1. Ensure everything passes
just full-check

# 2. Build and verify
just build-check

# 3. Test publish
just publish-test

# 4. Publish to PyPI
just publish
```

## CI Emulation

The `just ci` command emulates the GitHub Actions workflow exactly:

1. ✅ Pre-commit file checks (whitespace, EOF, YAML, etc.)
2. ✅ isort import sorting
3. ✅ Ruff linting
4. ✅ Ruff format checking
5. ✅ mypy type checking
6. ✅ Bandit security analysis
7. ✅ Markdown linting

This ensures your code will pass CI before you push.

## Tips

- Run `just` with no arguments to see all commands
- Use `just -l` or `just --list` for a detailed list
- Add custom recipes to the `justfile` for project-specific tasks
- Commands are organized by category with clear descriptions
